# Aurora — AI-Powered Health Companion

A full-stack agentic health companion app that helps users track hydration, sleep, habits, and nutrition — complete with an AI voice companion that automatically reviews and learns from your patterns.

---

## 🚀 Key Features & Custom Enhancements

### 1. Interactive AI Voice & Chat Companion
* **Natural Dialogue**: Speak or type to your health companion to discuss sleep habits, hydration, and general wellness.
* **Agent Actions**: The AI can write data back to the database! Tell it *"I drank 500ml water"* or *"I slept 8 hours"* and watch it instantly update your dashboard.
* **Request Context Fix**: All chat calls propagate `X-Mock-User-Email` headers, ensuring the backend successfully routes AI interactions to the active user profile database records.

### 2. Premium Mock Google & Apple OAuth Flows
* **Synchronous Session Updates**: Prevents race conditions during redirection by committing login states to `localStorage` immediately in the click event thread.
* **Glassmorphic Account Chooser**: Displays a modern, floating authentication card dialog when clicking "Continue with Google" or "Continue with Apple":
  * **Presets**: Quickly test with developer accounts (**Alex Mercer**, **Jane Doe**).
  * **Custom Option**: Enter any custom email and name to sign up or sign in as a new user.

### 3. Responsive Dark & Light Mode Toggles
* **Hydration-Safe Theme Toggle**: The landing page includes an interactive theme switch in the header, rendering custom icon sets (Amber `Sun` for dark mode, Indigo `Moon` for light mode).
* **Smooth Transitions**: Fully styled matching transitions on both the marketing pages and internal tracking dashboard.

### 4. Direct Client-side Navigation
* Wrapped all sidebar, mobile header, and landing page brand titles in wouter `Link` components to route home (`/`) without hard page reloads.

## 💻 Technical Stack

* **Monorepo Architecture**: `pnpm` workspaces (Node.js 24, TypeScript 5.9)
* **Frontend**: React 19, Vite 7, Tailwind CSS v4, shadcn/ui components, wouter client routing, Recharts dashboards
* **Backend API**: Express 5 (served at `/api`), esbuild compilation (ESM module bundle)
* **Database & ORM**: PostgreSQL 16 + Drizzle ORM, `schema.sql` initialization
* **Authentication**: Clerk cookie-based auth (with local mock fallback and interactive picker overlays)
* **AI Platform**: OpenAI API (gpt-4o-mini for messaging, Whisper-1 for speech transcription, TTS-1 for text-to-speech) & Gemini API
* **API Spec & Codegen**: OpenAPI YAML spec + Orval autogenerated React Query hooks and type definitions
* **Validation**: Zod v4 schema checks, `drizzle-zod` DB integrations

## 🛠️ Environment Variables Configuration

To run the application, configure your variables in the root [.env](file:///Users/bipasasaha/Documents/carebot/Untitled/AURORA-AI/.env) or [.env.local](file:///Users/bipasasaha/Documents/carebot/Untitled/AURORA-AI/.env.local) file:

```ini
# Auth Settings
BYPASS_CLERK=true

# AI Service Keys
OPENAI_API_KEY="your_openai_api_key"
GEMINI_API_KEY="your_gemini_api_key"

# Session Details
SESSION_SECRET="super-secret-session-key-12345"
```

## 💻 Run & Operate

### 1. Run Automatically (Recommended)
Simply launch the dev manager shell script from the root directory:
```bash
chmod +x run-dev.sh
./run-dev.sh
```
This script will load environment variables, check if PostgreSQL is running locally, apply all schema files, and launch both the backend (port `8080`) and frontend (port `8082`) concurrently.

### 2. Manual Commands
* **Run API Server**: `pnpm --filter @workspace/api-server run dev` (Port `8080`)
* **Run Frontend**: `pnpm --filter @workspace/aurora dev` (Port `8082`)
* **Typecheck Verification**: `pnpm --filter @workspace/aurora typecheck`
* **Codegen**: `pnpm --filter @workspace/api-spec run codegen`
