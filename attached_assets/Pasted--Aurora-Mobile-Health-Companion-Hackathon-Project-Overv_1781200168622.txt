# Aurora – Mobile Health Companion Hackathon

# Project Overview

Project Aurora is a mobile-first health companion designed to help users better understand themselves if they are healthy or unhealthy through hydration tracking, sleep tracking, habit building, nutrition awareness, personalized insights, and an intelligent health companion.

Most health applications focus on collecting data.

Project Aurora focuses on helping users understand what that data means and how they can use it to improve their daily lives.

The goal of this hackathon is to build a functional mobile prototype that demonstrates the core product experience of a modern AI-powered health companion.

The application should combine:

- Health Tracking
- Personalized Insights
- Voice AI
- Habit Formation
- Health Progress Monitoring

into a single experience that feels personal, intelligent, and enjoyable to use daily.

This is not about building a production-ready application.

This is about demonstrating:

- Product Thinking
- Mobile Development Skills
- UI/UX Design
- Backend Architecture
- AI Integration
- Execution Ability

---

# 🎯 What You Need to Build

Create a functional mobile application prototype for Project Aurora that includes the flows described below.

The application should feel like a personal health companion rather than a traditional health tracker.

The objective is to create an experience users would genuinely want to use every day.

# 📱 Application Flow

## 1. Intro / Landing Experience

When users open the application they should see:

"Understand yourself better every day."

Followed by onboarding screens introducing:

### Screen One

Meet your personal health companion.

### Screen Two

Track hydration, sleep, habits, and nutrition.

### Screen Three

Receive personalized daily insights.

### Screen Four

Build healthier routines through consistency.

### Screen Five

Learn more about yourself every day.

The experience should feel premium, modern, and calm.

**Note:** The examples and content provided below are intended as reference templates only and should not be treated as fixed requirements. Think creatively and make thoughtful product decisions that best align with the overall vision, user experience, and goals of the application.

## 10. 🎙️ Agentic Health Companion (Core Feature)

This is the most important feature of the assignment.

The companion should feel like a personal health coach rather than a chatbot.

### Voice-to-Voice Interaction (Required)

Users should be able to:

- Speak naturally
- Receive spoken responses
- Have conversations about their health

Examples:

"How am I doing this week?"

"Did I drink enough water today?"

"How can I improve my sleep?"

"What habits should I focus on?"

---

### Agent Actions (Required)

The AI should be capable of updating information through conversation.

Examples:

User:

"I drank 500ml water."

AI:

"Great. I've added 500ml to today's hydration progress."

User:

"I slept 7 hours last night."

AI:

"I've updated your sleep log."

User:

"Create a habit to meditate every morning."

AI:

"Done. I've created the habit."

The goal is to demonstrate an AI agent that can understand and take actions.

---

### Personalized Recommendations

The AI should use available user information to provide personalized suggestions whenever possible.

---

## 11. Health Memory System (Optional)
